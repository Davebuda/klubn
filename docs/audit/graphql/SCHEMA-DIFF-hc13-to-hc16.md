# GraphQL schema diff — HotChocolate 13.9.7 → 16.1.3 (WSx)

Generated by exporting `GET /graphql?sdl` before the upgrade (`schema.hc13.graphql`) and after
(`schema.hc16.graphql`), then diffing **semantically** (ignoring directive annotations, declaration
order, and whitespace/comma formatting). This file is the intentional-change record for the
HC13→16 migration.

## Verdict: no client-breaking schema change

- **48 object types before and after — identical set** (none added, none removed).
- **Every field is identical** in name, argument names, argument types, and nullability.
  The only field-level diff the raw text shows is cosmetic: HC16 emits commas between arguments
  (`approveDJApplication(applicationId: UUID!, reviewedByAdminId: String!)`) and wraps long
  argument lists across lines. No semantic change.
- **`Guid` still surfaces as `UUID!`** (141→142 occurrences — the +1 is formatting, not a new
  field). This is the load-bearing contract the SPA depends on; it is unchanged.

## Intentional changes (all explained / accepted)

| Change | HC13 | HC16 | Client impact |
|---|---|---|---|
| Cost directive | `directive @cost(complexity: Int!, multipliers: [MultiplierPath!], defaultMultiplier: Int)` (old **operation-complexity** analyzer) | `directive @cost(weight: …)` (new **cost analysis**) | Metadata only. Typed clients/codegen ignore unknown field directives. |
| Per-field annotation | none | **124 fields** annotated `@cost(weight: "10")` (the default field weight) | Appears in published SDL/introspection; additive, non-breaking. |
| `@listSize` | n/a | **0 usages** (schema has no cursor-paged fields) | None. |
| Scalar removed | `scalar MultiplierPath` | — | Removed because only the *old* `@cost` referenced it. No app field used it. |
| Scalars retained | `UUID, DateTime, Decimal, Long` | same | None. No scalar renamed (no `Url`→`URI`, `TimeSpan`→`Duration` etc. — those CLR types aren't in this schema). |
| Declaration order | `Query`/`Mutation` at end of SDL | `Query`/`Mutation` at top | Cosmetic ordering only. |

## What did NOT change (explicitly verified)
- No scalar renames affecting the wire (`UUID` stable; no `URI`/`Duration`/`Base64String` because
  the schema exposes none of those CLR types).
- No nullability changes on any field or argument.
- No new or removed root operations.
- Introspection content is unchanged in Development; it is **disabled outside Development**
  (verified live: `HC0046 "Introspection is not allowed"` in Staging).
